## 核心方法论

### Winston的成功公式

```
演讲质量 = f(知识, 练习, 天赋)
```

天赋占比极小。真正决定演讲质量的是：你掌握多少关于演讲的**知识**（启发式规则），以及你**练习**了多少次。

## 一、如何开场

### 不要以笑话开场

观众刚到场时还在适应——收手机、调整坐姿、适应你的声音。这时讲笑话极容易冷场，而冷场会影响你整场的信心。

### 用「赋能承诺」开场（Empowerment Promise）

告诉观众：**听完这段，你将获得什么**。给他们一个留下来的理由。

**好的承诺：**
- 「接下来30分钟，你会学到3个可以今天就用的AI编程技巧」
- 「听完这段，你会知道为什么90%的人用AI工具的方式都是错的」

**差的承诺：**
- 「今天我来分享一下AI编程的经验」（太模糊，没有价值感）
- 「大家好，我是花叔，很高兴来到这里」（废话开场）

### 要求全注意力

人类只有一个语言处理器。如果观众在看手机，他们**听不进去**你说的话。更糟的是，看手机的人会分散旁边人的注意力，也会让演讲者分心。

**线下培训时**：直接要求合上电脑、放下手机
**视频场景**：前10秒必须给出足够强的理由让观众留下

## 二、演讲过程的四个核心技巧

### 1. 循环往复（Cycling）

在任何时刻，约20%的观众处于走神状态。通过3次重复核心概念（用不同方式），确保每个人都能接收到关键信息。

**操作方法**：「告诉他们你要讲什么 → 讲给他们听 → 告诉他们你讲了什么」

**视频场景应用**：每隔3-5分钟回扣主题，用一句话把观众拉回来。「所以回到我们的核心问题——为什么手搓经济突然火了？」

### 2. 筑墙划界（Build a Fence）

明确你的想法与他人想法的**区别**，防止观众混淆。

**操作方法**：说清楚「我讲的是什么」和「我讲的不是什么」

**示例**：
- 「我说的AI编程不是让AI帮你写Hello World，而是用AI从0到1做一个完整产品」
- 「这跟普通的Prompt Engineering不同，区别在于...」

### 3. 口头标点（Verbal Punctuation）

为走神的观众提供「重新上车」的机会。用明确的结构提示，让迷失的观众知道你讲到了哪里。

**操作方法**：
- 「好，第一点我们讲完了。现在进入第二点...」
- 「到这里我们已经讲了A和B，接下来是最关键的C...」
- 使用编号：「第一、第二、第三...」

### 4. 提问与等待（Ask Questions）

提问能瞬间抓回注意力。关键是提问后**必须等待**。

**等待时间**：至少7秒。不要因为沉默尴尬就自己回答。

**问题难度**：
- 太简单 → 观众觉得被侮辱
- 太难 → 没人敢回答
- 刚好 → 需要想一下但能答出来

**视频场景**：虽然观众无法真正回答，但提问制造了思考的停顿，比直接给答案更有效。

## 三、时间与地点

| 规则 | 要求 | 原因 |
|------|------|------|
| 最佳时间 | 上午11点 | 大家醒了，还没午饭犯困 |
| 灯光 | 全开 | 灯暗=睡觉信号。「闭着眼皮看不清幻灯片」 |
| 踩点 | 提前到场检查 | 了解设备、灯光开关、话筒、座位布局 |
| 填充房间 | 选会被填满一半以上的房间 | 人太少气氛冷清，宁可换小房间 |

## 四、工具与道具

### 黑板/白板的优势

1. **图形化**：方便即时画图解释
2. **节奏控制**：写字的速度正好匹配观众吸收速度（幻灯片往往太快）
3. **手的归宿**：给手找事做，避免插口袋或背后的尴尬姿态
4. **共情镜像**：观众会心理模拟你写字的动作，形成更深的参与感

### 道具的力量

道具有极强的视觉记忆点。观众可能忘了理论，但一定记住道具。

**应用**：演讲中涉及实物时，拿出来展示。比如讲「小猫补光灯」时，真的拿出手机演示。

### 幻灯片的七宗罪

| 罪行 | 为什么有问题 | 怎么改 |
|------|-------------|--------|
| 朗读幻灯片 | 观众识字比你读得快，会烦 | 幻灯片是提示，不是台词 |
| 离屏幕太远 | 观众像看网球，视线来回跳 | 站在屏幕旁边 |
| 堆砌文字 | 信息过载，什么都记不住 | 字体≥40号，倒逼精简 |
| 使用剪贴画/Logo | 视觉噪音 | 删掉一切装饰 |
| 使用激光笔 | 背对观众，失去眼神交流 | 在幻灯片上画箭头，或用手指 |
| 复杂背景图案 | 降低可读性 | 纯色背景 |
| 要点过多 | 每张太密 | 一张一个核心观点 |

### 唯一例外：Hapax Legomenon

整场演讲中允许**一张**极其复杂的幻灯片。目的不是让观众看清内容，而是展示问题的**复杂性和规模**，建立威信。

## 五、如何启发（Inspire）

### 对新手/学生

传递「你行的」的信念比教具体技能更重要。让他们相信自己能做到。

### 对有经验的人

展示你对这件事的**真实热情**。如果你觉得某个东西很酷，就表现出来。激情是最强的感染力。

### 教人思考而非教人知识

Winston主张通过**故事**来教思考方式。提供：
- 故事（案例）
- 关于故事的关键问题
- 分析框架
- 综合方法
- 可靠性评估工具

## 六、如何结尾

### 绝对不要做的事

- ❌ 最后一页写「谢谢」「Questions?」「The End」— 浪费屏幕
- ❌ 说「感谢大家的时间」— 暗示观众是出于礼貌才听的，把自己放在弱势位置

### 应该怎么做

1. **明确发出结束信号**：让观众知道你要结尾了
2. **最后一页放「贡献」**：列出你的核心成果/要点。问答环节可能持续20分钟，观众会一直看着你的成就
3. **致敬观众（Salute）**：不是感谢，是致敬。肯定他们来到这里的价值
4. **结尾的笑话可以**：与「不以笑话开场」不矛盾。此时观众已经适应你了，笑话是调味剂

**好的结尾示例**：
- 「这就是今天分享的全部内容。你们愿意花这个时间来了解AI编程，说明你们是真正想做点什么的人。期待看到你们的作品。」
- 「我希望今天的分享能让你们少走一些弯路。下次有机会再聊。」

## 七、Winston之星：让你的工作被记住

如果你希望自己的项目/产品/研究被人记住，需要5个要素：

| 要素 | 说明 | 花叔示例 |
|------|------|---------|
| **符号 Symbol** | 一个可视化图标 | 小猫补光灯的猫咪图标 |
| **口号 Slogan** | 一个朗朗上口的短语 | 「手搓经济」「Vibe Coding」 |
| **惊喜 Surprise** | 一个反直觉的事实 | 「开发成本只有几十块钱API费」 |
| **突出的想法 Salient Idea** | 最容易被记住的概念 | 「一人公司」 |
| **故事 Story** | 它是怎么发生的 | 小猫补光灯从自用到付费榜第一的经历 |

## 八、隐性技巧（Meta-techniques）

Winston通过自己的演讲**演示**但未明确教授的技巧：

1. **知行合一**：他严格遵守自己的每一条规则（没用激光笔、灯光全开、使用黑板、带道具）
2. **利用沉默**：在关键时刻故意停顿，让观众自己思考，而不是填满每一秒
3. **冷幽默**：自然穿插的幽默是调味剂，与「不开场讲笑话」不矛盾
4. **移动控场**：不在讲台后静止，在舞台上移动，用位置变化引导注意力
5. **身体语言**：手势指向黑板/屏幕，眼神扫视全场，与观众建立连接

## 如何使用这个Skill

### 场景1：审查演讲/培训大纲

给我你的演讲大纲，我会从以下维度审查：
- [ ] 有没有赋能承诺？
- [ ] 核心观点是否循环了3次？
- [ ] 有没有筑墙划界？
- [ ] 口头标点是否充分？
- [ ] 有没有互动提问？
- [ ] 结尾是否有力（非「谢谢」）？
- [ ] Winston之星5要素是否具备？

### 场景2：优化视频脚本

给我你的视频脚本，我会检查：
- 前10秒的钩子（赋能承诺）
- 每3-5分钟是否有回扣主题的循环点
- 口头标点的密度是否够
- 节奏变化（信息密集段 vs 喘息段）
- 结尾是否给出行动引导

### 场景3：PPT/Keynote审查

给我你的幻灯片，我会逐张检查七宗罪：
- 文字量是否过多（字号≥40？）
- 是否有装饰性元素（Logo/剪贴画）
- 是否有复杂背景
- 信息层次是否清晰
- 是否有一张「Hapax Legomenon」复杂图

### 场景4：模拟观众反馈

我会模拟一个具体观众画像（如：「第一次接触AI编程的产品经理」），从他们的视角听你的演讲，标记：
- 哪里会走神
- 哪里会产生疑问
- 哪里会感到兴奋
- 哪里信息过载

---

## 审查评分（10分制）

当审查演讲/脚本时，从5个维度评分：

| 维度 | 评判标准 |
|------|---------|
| **开场力** | 赋能承诺的吸引力、前60秒的抓取力 |
| **结构清晰度** | 循环、口头标点、筑墙划界是否到位 |
| **互动性** | 提问、停顿、观众参与设计 |
| **视觉辅助** | 幻灯片/道具是否遵守规则 |
| **结尾力** | 是否有力、是否给出贡献/行动、是否致敬而非感谢 |

综合评分 + 逐项建议 + 优先修改项（改动最小、收益最大的3件事）

## 速查清单

**演讲前**：
- [ ] 赋能承诺写好了吗？
- [ ] 踩点了吗？灯光够亮吗？
- [ ] 房间大小合适吗？
- [ ] 幻灯片过了七宗罪检查吗？
- [ ] 练习了至少1遍吗？

**演讲中**：
- [ ] 开场不讲笑话，直接给承诺
- [ ] 核心观点至少循环3次
- [ ] 每5-10分钟有口头标点
- [ ] 提问后等待7秒
- [ ] 不读幻灯片，不用激光笔

**演讲后**：
- [ ] 最后一页是贡献，不是「谢谢」
- [ ] 致敬观众，不是感谢观众
- [ ] 结尾有行动引导或金句
